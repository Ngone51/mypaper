20180323 星期五 备忘录
截止到今天，我们GNMF的大致做法是：
准备一个普通用户和大V用户之间的关注矩阵F；
准备所有用户（包括普通用户和大V用户）之间的关注图G(N, E)，然后通过node2vec，得到所有用户的特征向量。
然后再从中抽取其中大V用户的特征向量，用于构建相似度矩阵W。
我们应用GNMF，把关注矩阵F和相似度矩阵W融合起来，获取分解结果，然后重构矩阵，做推荐。

注意：但是这里有一个问题，可能从根本上来说就是错误的。那就是，我们的关注图G(N, E)包含了除了我们
关注矩阵F中的用户信息，并应用到了GNMF上。而这部分信息，其它的推荐方法NMF、SVD是没有的！也就说，不同的
方法之间的训练集是不一样的。

所以，我们计算W的方式需要重新设计。虽然，在原有的代码框架上，真正要修改代码的地方可能不多，但是对于整个
推荐模型的设计来说，却是一个大的改动！这直接影响最终推荐模型的推荐效果。
具体的改动：我们需要用相同的用户数据来分别构建关注矩阵F和关注图G（N，E）。在关注矩阵F中有的用户，在关注
图G中也必须有，而在关注矩阵F中没有的用户，在关注图G中也必须没有。但是，在关注图G中有的关注关系，在关注矩阵F
中不一定会有。因为，在关注矩阵F中，只有普通用户到大V用户的关注关系。而在关注图G中，可能存在普通用户和普通用户
之间，大V用户和大V用户之间，以及大V用户到普通用户的关注关系。

注意：新的和计算W相关的文件名末尾以“2”标记。
